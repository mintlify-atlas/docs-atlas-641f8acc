---
title: Authorization API
description: Manage roles, permissions, and authorization checks
---

The Authorization API provides a comprehensive role-based access control (RBAC) system with support for environment roles, organization roles, permissions, resources, and role assignments.

## Initialize

```typescript
import { WorkOS } from '@workos-inc/node';

const workos = new WorkOS('sk_example_123456789');
```

## Environment Roles

Environment roles are global roles that apply across your entire WorkOS environment.

### createEnvironmentRole

Create a new environment role.

```typescript
const role = await workos.authorization.createEnvironmentRole({
  slug: 'admin',
  name: 'Administrator',
  description: 'Full system access',
  resourceTypeSlug: 'project',
  permissions: ['read', 'write', 'delete'],
});
```

<ParamField path="options" type="CreateEnvironmentRoleOptions" required>
  <Expandable title="properties">
    <ParamField path="slug" type="string" required>
      Unique identifier for the role.
    </ParamField>
    <ParamField path="name" type="string" required>
      Display name of the role.
    </ParamField>
    <ParamField path="description" type="string" optional>
      Description of the role.
    </ParamField>
    <ParamField path="resourceTypeSlug" type="string" required>
      Type of resource this role applies to.
    </ParamField>
    <ParamField path="permissions" type="string[]" optional>
      Array of permission slugs.
    </ParamField>
  </Expandable>
</ParamField>

<ResponseField name="role" type="EnvironmentRole">
  The created environment role.
</ResponseField>

## Authorization Checks

### check

Check if a user has a specific permission on a resource.

```typescript
const result = await workos.authorization.check({
  organizationMembershipId: 'om_123',
  permissionSlug: 'read',
  resourceId: 'resource_123',
});

console.log(result.authorized); // true or false
```

<ParamField path="options" type="AuthorizationCheckOptions" required>
  <Expandable title="properties">
    <ParamField path="organizationMembershipId" type="string" required>
      The organization membership ID to check.
    </ParamField>
    <ParamField path="permissionSlug" type="string" required>
      The permission to check for.
    </ParamField>
    <ParamField path="resourceId" type="string" optional>
      The resource ID (use this OR externalId).
    </ParamField>
    <ParamField path="resourceExternalId" type="string" optional>
      The resource external ID.
    </ParamField>
    <ParamField path="resourceTypeSlug" type="string" optional>
      The resource type slug (required with resourceExternalId).
    </ParamField>
  </Expandable>
</ParamField>

<ResponseField name="result" type="AuthorizationCheckResult">
  <Expandable title="properties">
