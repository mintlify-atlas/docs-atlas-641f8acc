</Warning>

The Fine-Grained Authorization (FGA) API provides a flexible authorization system based on relationship-based access control (ReBAC). This API is deprecated and maintained for backward compatibility only.

## Initialize

```typescript
import { WorkOS } from '@workos-inc/node';

const workos = new WorkOS('sk_example_123456789');
```

## Methods

### check

Check if a subject has a specific relation to a resource.

```typescript
const result = await workos.fga.check({
  checks: [
    {
      resource: { resourceType: 'document', resourceId: 'doc_123' },
      relation: 'viewer',
      subject: { resourceType: 'user', resourceId: 'user_456' },
    },
  ],
});

console.log(result.isAuthorized());
```

<ParamField path="checkOptions" type="CheckOptions" required>
  <Expandable title="properties">
    <ParamField path="op" type="CheckOp" optional>
      Operator for combining multiple checks (`and`, `or`).
    </ParamField>
    <ParamField path="checks" type="CheckWarrantOptions[]" required>
      Array of warrant checks to perform.
    </ParamField>
    <ParamField path="debug" type="boolean" optional>
      Enable debug mode for detailed decision tree.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField path="options" type="CheckRequestOptions" optional>
  <Expandable title="properties">
    <ParamField path="warrantToken" type="string" optional>
      Warrant token for consistent reads.
    </ParamField>
  </Expandable>
</ParamField>

<ResponseField name="result" type="CheckResult">
  <Expandable title="properties">
    <ResponseField name="result" type="string">
      Result of the check.
    </ResponseField>
    <ResponseField name="isImplicit" type="boolean">
      Whether the authorization is implicit.
    </ResponseField>
    <ResponseField name="warrantToken" type="string">
      Token for consistent reads.
    </ResponseField>
    <ResponseField name="isAuthorized()" type="() => boolean">
      Helper method to check if authorized.
    </ResponseField>
  </Expandable>
</ResponseField>

### checkBatch

Perform multiple independent authorization checks.

```typescript
const results = await workos.fga.checkBatch({
  checks: [
    {
      resource: { resourceType: 'document', resourceId: 'doc_123' },
      relation: 'viewer',
      subject: { resourceType: 'user', resourceId: 'user_456' },
    },
    {
      resource: { resourceType: 'document', resourceId: 'doc_789' },
      relation: 'editor',
      subject: { resourceType: 'user', resourceId: 'user_456' },
    },
  ],
});

results.forEach((result) => {
  console.log(result.isAuthorized());
});
```

<ParamField path="checkOptions" type="CheckBatchOptions" required>
  <Expandable title="properties">
    <ParamField path="checks" type="CheckWarrantOptions[]" required>
      Array of warrant checks to perform.
    </ParamField>
    <ParamField path="debug" type="boolean" optional>
      Enable debug mode.
    </ParamField>
  </Expandable>
</ParamField>

<ResponseField name="results" type="CheckResult[]">
  Array of check results, one for each check.
</ResponseField>

## Migration Guide

To migrate from FGA to the Authorization API:

1. Replace `workos.fga.check()` with `workos.authorization.check()`
2. Replace resource management with `workos.authorization.createResource()`, etc.
3. Use `workos.authorization.assignRole()` instead of warrant operations
4. See the [Authorization API documentation](/api/authorization) for complete details

## Example: Access Control with FGA

```typescript
// Create resources
await workos.fga.createResource({
  resource: { resourceType: 'document', resourceId: 'doc_123' },
  meta: { name: 'Confidential Report' },
});

// Grant viewer access
await workos.fga.writeWarrant({
  resource: { resourceType: 'document', resourceId: 'doc_123' },
  relation: 'viewer',
  subject: { resourceType: 'user', resourceId: 'user_456' },
});

// Check access
const result = await workos.fga.check({
  checks: [
    {
      resource: { resourceType: 'document', resourceId: 'doc_123' },
      relation: 'viewer',
      subject: { resourceType: 'user', resourceId: 'user_456' },
    },
  ],
});

if (result.isAuthorized()) {
  console.log('Access granted');
}
```
