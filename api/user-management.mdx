title: "UserManagement API"
description: "Manage users, authentication, sessions, and organization memberships"

The `UserManagement` class provides comprehensive methods for user authentication, user lifecycle management, session handling, and organization membership management.

## User Methods

### getUser

Retrieves a user by their ID.

<ParamField path="userId" type="string" required>
  The unique identifier of the user
</ParamField>

<ResponseField name="user" type="User">
  <ResponseField name="id" type="string">
    Unique identifier for the user
  </ResponseField>
  <ResponseField name="email" type="string">
    User's email address
  </ResponseField>
  <ResponseField name="emailVerified" type="boolean">
    Whether the user's email has been verified
  </ResponseField>
  <ResponseField name="firstName" type="string | null">
    User's first name
  </ResponseField>
  <ResponseField name="lastName" type="string | null">
    User's last name
  </ResponseField>
  <ResponseField name="profilePictureUrl" type="string | null">
    URL to user's profile picture
  </ResponseField>
  <ResponseField name="createdAt" type="string">
    ISO 8601 timestamp of user creation
  </ResponseField>
  <ResponseField name="updatedAt" type="string">
    ISO 8601 timestamp of last update
  </ResponseField>
  <ResponseField name="externalId" type="string | null">
    External identifier from your system
  </ResponseField>
  <ResponseField name="metadata" type="Record<string, string>">
    Custom metadata associated with the user
  </ResponseField>
</ResponseField>

```typescript
const user = await workos.userManagement.getUser('user_01HZXK8F9P3QZJQ2Z1Z2Z3Z4Z5');
console.log(user.email);
```

## Authentication Methods

### authenticateWithPassword

Authenticates a user with email and password.

<ParamField body="email" type="string" required>
  User's email address
</ParamField>

<ParamField body="password" type="string" required>
  User's password
</ParamField>

<ParamField body="clientId" type="string">
  OAuth client ID (optional if configured globally)
</ParamField>

<ParamField body="session" type="AuthenticateWithSessionOptions">
  Session options for sealing the session
</ParamField>

<ResponseField name="authenticationResponse" type="AuthenticationResponse">
  <ResponseField name="user" type="User">
    The authenticated user
  </ResponseField>
  <ResponseField name="accessToken" type="string">
    JWT access token
  </ResponseField>
  <ResponseField name="refreshToken" type="string">
    Refresh token for obtaining new access tokens
  </ResponseField>
  <ResponseField name="organizationId" type="string">
    ID of the user's organization
  </ResponseField>
  <ResponseField name="authenticationMethod" type="string">
    Method used for authentication (e.g., 'Password')
  </ResponseField>
  <ResponseField name="sealedSession" type="string">
    Encrypted session data (if sealSession option was enabled)
  </ResponseField>
</ResponseField>

```typescript
const authResponse = await workos.userManagement.authenticateWithPassword({
  email: 'user@example.com',
  password: 'securePassword123',
  clientId: 'client_01HZXK8F9P3QZJQ2Z1Z2Z3Z4Z5'
});

console.log(authResponse.accessToken);
```

### authenticateWithCode

Exchanges an authorization code for tokens. Auto-detects public vs confidential client mode based on whether a code verifier or API key is provided.

<ParamField body="code" type="string" required>
  Authorization code from OAuth callback
</ParamField>

<ParamField body="clientId" type="string">
  OAuth client ID (optional if configured globally)
</ParamField>

<ParamField body="codeVerifier" type="string">
  PKCE code verifier (for public clients)
</ParamField>

<ParamField body="session" type="AuthenticateWithSessionOptions">
  Session options for sealing the session
</ParamField>

<ResponseField name="authenticationResponse" type="AuthenticationResponse">
  Authentication response with user and tokens
</ResponseField>

```typescript
// Public client (PKCE flow)
const authResponse = await workos.userManagement.authenticateWithCode({
  code: 'auth_code_123',
  codeVerifier: 'code_verifier_from_pkce',
  clientId: 'client_01HZXK8F9P3QZJQ2Z1Z2Z3Z4Z5'
});

// Confidential client (uses API key)
const authResponse = await workos.userManagement.authenticateWithCode({
  code: 'auth_code_123',
  clientId: 'client_01HZXK8F9P3QZJQ2Z1Z2Z3Z4Z5'
});
```

### getAuthorizationUrl

Generates an OAuth 2.0 authorization URL.

<ParamField body="provider" type="string">
  OAuth provider (e.g., 'authkit', 'GoogleOAuth')
</ParamField>

<ParamField body="connectionId" type="string">
  SSO connection ID
</ParamField>

<ParamField body="organizationId" type="string">
  Organization ID
</ParamField>

<ParamField body="redirectUri" type="string" required>
  URL to redirect after authorization
</ParamField>

<ParamField body="state" type="string">
  State parameter for CSRF protection
</ParamField>

<ParamField body="codeChallenge" type="string">
  PKCE code challenge
</ParamField>

<ParamField body="codeChallengeMethod" type="string">
  PKCE code challenge method (e.g., 'S256')
</ParamField>

<ParamField body="screenHint" type="string">
  Screen hint for AuthKit (e.g., 'sign-up', 'sign-in')
</ParamField>

<ResponseField name="url" type="string">
  The authorization URL
</ResponseField>

```typescript
const url = workos.userManagement.getAuthorizationUrl({
  provider: 'authkit',
  redirectUri: 'https://myapp.com/callback',
  state: 'random_state_123',
  screenHint: 'sign-up'
});

// Redirect user to this URL
window.location.href = url;
```

### getAuthorizationUrlWithPKCE

Generates an OAuth 2.0 authorization URL with automatic PKCE generation.

<ParamField body="provider" type="string">
  OAuth provider (e.g., 'authkit')
</ParamField>

<ParamField body="connectionId" type="string">
  SSO connection ID
</ParamField>

<ParamField body="organizationId" type="string">
  Organization ID
</ParamField>

<ParamField body="redirectUri" type="string" required>
  URL to redirect after authorization
</ParamField>

<ParamField body="screenHint" type="string">
  Screen hint for AuthKit
</ParamField>

<ResponseField name="result" type="PKCEAuthorizationURLResult">
  <ResponseField name="url" type="string">
    The authorization URL
  </ResponseField>
  <ResponseField name="state" type="string">
    Generated state parameter
  </ResponseField>
  <ResponseField name="codeVerifier" type="string">
    Code verifier for PKCE (store securely)
  </ResponseField>
</ResponseField>

```typescript
const { url, state, codeVerifier } = await workos.userManagement.getAuthorizationUrlWithPKCE({
  provider: 'authkit',
  clientId: 'client_01HZXK8F9P3QZJQ2Z1Z2Z3Z4Z5',
  redirectUri: 'myapp://callback'
});

// Store state and codeVerifier securely
localStorage.setItem('pkce_state', state);
localStorage.setItem('pkce_verifier', codeVerifier);

// Redirect user
window.location.href = url;
```

## Session Methods

### loadSealedSession

Loads a sealed session for session management.

<ParamField body="sessionData" type="string" required>
  Encrypted session data from cookie
</ParamField>

<ParamField body="cookiePassword" type="string" required>
  Password used to encrypt the session
</ParamField>

<ResponseField name="session" type="CookieSession">
  Session object for managing the session
</ResponseField>

```typescript

  sessionData: req.cookies.session,
  cookiePassword: process.env.WORKOS_COOKIE_PASSWORD
});

```

### authenticateWithSessionCookie

Authenticates and validates a session cookie.

<ParamField body="sessionData" type="string" required>
  Encrypted session data from cookie
</ParamField>

<ParamField body="cookiePassword" type="string">
  Password used to encrypt the session (defaults to WORKOS_COOKIE_PASSWORD env var)
</ParamField>

<ResponseField name="result" type="AuthenticateWithSessionCookieSuccessResponse | AuthenticateWithSessionCookieFailedResponse">
  <ResponseField name="authenticated" type="boolean">
    Whether authentication was successful
  </ResponseField>
  <ResponseField name="user" type="User">
    The authenticated user (if successful)
  </ResponseField>
  <ResponseField name="sessionId" type="string">
    Session ID (if successful)
  </ResponseField>
  <ResponseField name="organizationId" type="string">
    Organization ID (if successful)
  </ResponseField>
  <ResponseField name="reason" type="string">
    Failure reason (if failed)
  </ResponseField>
</ResponseField>

```typescript
const result = await workos.userManagement.authenticateWithSessionCookie({
  sessionData: req.cookies.session,
  cookiePassword: process.env.WORKOS_COOKIE_PASSWORD
});

if (result.authenticated) {
  console.log('User:', result.user.email);
} else {
  console.log('Auth failed:', result.reason);
}
```

### enrollAuthFactor

Enrolls a new authentication factor (MFA).

  The user ID
</ParamField>

<ParamField body="type" type="'totp' | 'sms'" required>

</ParamField>

  Phone number (required for SMS type)
</ParamField>

<ParamField body="issuer" type="string">
  TOTP issuer (required for TOTP type)
</ParamField>

<ParamField body="user" type="string">
  TOTP user (required for TOTP type)
</ParamField>

<ResponseField name="result" type="object">
  <ResponseField name="authenticationFactor" type="FactorWithSecrets">
    The enrolled factor with secrets
  </ResponseField>
  <ResponseField name="authenticationChallenge" type="Challenge">
    Initial authentication challenge
  </ResponseField>
</ResponseField>

```typescript

  await workos.userManagement.enrollAuthFactor({
    userId: 'user_01HZXK8F9P3QZJQ2Z1Z2Z3Z4Z5',
    type: 'totp',
    issuer: 'MyApp',
```
